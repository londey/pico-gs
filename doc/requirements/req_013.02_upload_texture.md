# REQ-013.02: Upload Texture

## Requirement

When an UploadTexture command is dispatched, the system SHALL write the texture index array to GPU SDRAM via the MEM_ADDR/MEM_DATA auto-increment register protocol, starting at the target SDRAM address specified in the command, and SHALL complete the upload before issuing any subsequent triangle submission commands that reference that texture.

## Rationale

Texture data must be resident in GPU SDRAM before any triangle that references it is submitted.
The MEM_ADDR/MEM_DATA auto-increment protocol (INT-010) provides the bulk upload path over the SPI register interface.

## Parent Requirements

REQ-013 (Host SPI Driver)

## Interfaces

- INT-010 (GPU Register Map) — MEM_ADDR/MEM_DATA registers
- INT-011 (SDRAM Memory Layout) — texture region address constraints
- INT-014 (Texture Memory Layout) — per-format block organization and size

## Verification Method

**Test:** Verify that an UploadTexture command produces the expected sequence of MEM_ADDR write followed by MEM_DATA writes for a known texture payload, with the correct target SDRAM address.
Verify that no triangle submission register writes occur between the MEM_ADDR setup and the final MEM_DATA write of the upload sequence.

## Notes

The GPU uses a single texture format, INDEXED8_2X2 (TEXn_CFG.FORMAT = 4'd0): an 8-bit palette index per 2×2 apparent-texel tile stored in a 4×4-block-tiled SDRAM layout (see INT-014).
Two distinct upload payloads exist and must be handled separately by the host driver:

**1. Index array upload (per-texture):**
The UploadTexture command writes the INDEXED8_2X2 index array to SDRAM via the MEM_ADDR/MEM_DATA auto-increment protocol.
Each 4×4 block covers 8×8 apparent texels (indices are at half resolution); a 64×64 apparent-texel texture therefore stores 32×32 = 1024 index bytes in SDRAM.
The upload sequence is byte-sequential; the host writes raw index bytes via MEM_DATA.
The upload must complete before any triangle that references the texture's TEXn_CFG.BASE_ADDR is submitted.

**2. Palette blob upload (per palette slot):**
The palette codebook is uploaded separately using a MEM_ADDR/MEM_DATA write to place the 4096-byte RGBA8888 blob at the SDRAM address encoded in PALETTEn.BASE_ADDR (byte address = BASE_ADDR × 512), followed by a write to PALETTEn.LOAD_TRIGGER.
The LOAD_TRIGGER write causes the GPU to DMA-load the blob from SDRAM into the on-chip palette EBR with inline UNORM8→UQ1.8 promotion.
Firmware must complete the SDRAM blob write and issue the LOAD_TRIGGER write before submitting any triangle whose texture references that palette slot via TEXn_CFG.PALETTE_IDX.
Because there is no hardware load-completion status register (REQ-003.09), firmware enforces ordering by not issuing dependent triangle submissions until sufficient cycles have elapsed or by structuring the command sequence such that palette load precedes all dependent draws.

Texture index array layout in SDRAM must conform to INT-011 (SDRAM Memory Layout) and INT-014 (Texture Memory Layout).
