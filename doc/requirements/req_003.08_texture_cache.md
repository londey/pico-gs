# REQ-003.08: Texture Cache

## Requirement

When the texture sampler (UNIT-011) samples textures during rendering, the system SHALL cache 8-bit palette indices in a per-sampler index cache to reduce external SDRAM bandwidth and enable single-cycle index access.

## Rationale

Texture sampling is memory-intensive.
Without caching, every pixel would perform an SDRAM read for its index block.
A per-sampler index cache exploits spatial locality: adjacent pixels often sample the same 4×4 apparent-texel block, which covers a 2×2 index tile, achieving high cache hit rates at low EBR cost.
Independent caches for each of the 2 texture samplers eliminate stalls from dual-texture rendering.
The palette LUT (UNIT-011.06) is resident in EBR and requires no caching.

**Performance impact (at 100 MHz `clk_core`):**

- Cache hit: 1 cycle / 10 ns for the index
- Cache miss: SDRAM fetch of a 16-byte index block (4×4 apparent texels, 2×2 index tiles), including CAS latency (CL=3) and row activation overhead
- Best-case fragment latency: 4 cycles (index cache hit → palette LUT read → Q4.12 promotion → output)

**Resource cost:**

- 2 independent per-sampler index caches, 1× DP16KD each (2 EBR total)
- 1 shared palette LUT, 4× DP16KD (4 EBR total — slot-banked, two 1024×18 DP16KDs in parallel per slot)
- 6 EBR total for the texture subsystem
- ~750 LUT4 for cache control, tag comparison, and palette load FSM
- 0 DSP

## Functional Requirements

### FR-131-1: Per-Sampler Index Cache Independence

When multiple texture samplers are active, each of the 2 per-sampler index caches SHALL operate independently with no cross-sampler contention or shared state.

### FR-131-2: Index Cache Capacity

When the system is configured, each per-sampler index cache SHALL store 32 sets × 1 way × 16 indices per line = 512 index entries of 8 bits each, using a single DP16KD EBR bank.

### FR-131-3: Single-Cycle Index Access

When a texture sampling operation requires a palette index and it is cached, the index cache SHALL provide the 8-bit index in a single clock cycle.

### FR-131-4: Index Cache Invalidation

When firmware writes to a TEXn_CFG register, the system SHALL invalidate the corresponding sampler's index cache to prevent stale index access.
Invalidating sampler N's index cache SHALL NOT affect sampler M (N ≠ M).
The shared palette LUT is NOT invalidated by TEXn_CFG writes; palette slots are managed independently via PALETTEn registers.

### FR-131-5: Index Storage Format

When indices are stored in the index cache, the system SHALL store them as raw 8-bit unsigned values (one index per apparent 2×2 texel tile) fetched directly from the SDRAM index array without transformation.

### FR-131-6: EBR Budget Compliance

The total EBR allocation for the texture cache subsystem (index caches + palette LUT) across all GPU components SHALL remain within the ECP5-25K budget (see REQ-011.02).

### FR-131-7: Shared Palette LUT

The system SHALL provide a single palette LUT (UNIT-011.06) shared between both texture samplers, storing 2 palette slots × 256 entries × 4 quadrant UQ1.8 RGBA colors.
Cross-slot sampler accesses SHALL proceed in parallel without contention; same-slot accesses MAY serialize via a sampler-priority mux on the shared read port (see UNIT-011.06 §Future Enhancements for the deferred dual-port enablement that removes this restriction).

## Verification Method

**Test:** Verify cache behavior meets the following acceptance criteria:

- [ ] Each of the 2 per-sampler index caches operates independently with no cross-sampler contention
- [ ] Each index cache stores 512 8-bit index entries (32 sets × 16 indices/line)
- [ ] Index read returns correct 8-bit value in 1 cycle on cache hit
- [ ] Index cache miss triggers SDRAM fetch of the 16-byte block and correct cache line fill
- [ ] Direct-mapped (1-way) set replacement verified: two blocks mapping to the same set cause expected eviction
- [ ] XOR set indexing distributes adjacent blocks across different sets (no systematic aliasing)
- [ ] TEXn_CFG write invalidates sampler N's index cache (next access is a guaranteed miss)
- [ ] Invalidating sampler 0's index cache does not affect sampler 1 and vice versa
- [ ] Stale indices never served after TEXn_CFG configuration change
- [ ] Shared palette LUT serves cross-slot sampler reads in parallel; same-slot reads serialize correctly under sampler-priority mux
- [ ] Total EBR allocation ≤ 6 blocks (2 index DP16KD + 4 palette DP16KD)

## Design Allocation

- UNIT-011.03 (Index Cache) — per-sampler DP16KD single-bank 8-bit index store, direct-mapped, XOR set indexing, SDRAM fill FSM (FR-131-1 through FR-131-5)
- UNIT-011.06 (Palette LUT) — shared DP16KD 2-slot codebook (1024×36 logical per slot, two 1024×18 DP16KDs in parallel), UQ1.8 RGBA storage, SDRAM load FSM (FR-131-6, FR-131-7)

## Notes

The index cache uses a single DP16KD per sampler (half-resolution: one 8-bit index covers a 2×2 apparent-texel tile).
The SDRAM index array retains the INT-014 4×4-block tiling convention; each block covers 8×8 apparent texels (2×2 index tiles per block, 4 blocks per cache line).
The palette LUT is resident in 4 DP16KD EBR banks (slot-banked: two 1024×18 DP16KDs in parallel per slot, providing one logical 1024×36 dual-port memory per slot) and does not require a separate cache level.
Detailed architecture (line formats, EBR allocation, invalidation protocol, set indexing) is specified in UNIT-011.03 and UNIT-011.06.
See DD-010 in design_decisions.md for per-sampler cache rationale.
