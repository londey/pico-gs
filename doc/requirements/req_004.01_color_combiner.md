# REQ-004.01: Color Combiner

## Classification

- **Priority:** Important
- **Stability:** Draft
- **Verification:** Test

## Requirement

When the pixel pipeline produces a fragment during rasterization, the system SHALL route that fragment through a three-pass programmable color combiner that accepts texture colors, vertex colors, constant colors, framebuffer destination color, and a combined-output feedback path as inputs and produces a single combined output color including alpha blending.

## Rationale

A three-pass `(A-B)*C+D` color combiner (CC0, CC1, CC2/blend) replaces the previous two-stage combiner plus dedicated alpha blend unit.
Each pass independently combines available input sources for RGB and Alpha using a single time-multiplexed datapath.
Pass 0's output feeds pass 1, and pass 1's output feeds pass 2, via the COMBINED source.
Pass 2 introduces DST_COLOR (promoted destination pixel from the color tile buffer), expressing all alpha blend modes as CC equations and eliminating the need for a separate blend unit.
This architecture is inspired by the N64 RDP combiner, extended with a blend pass and adapted to the Q4.12 pipeline format.

## Functional Requirements

### FR-009-1: Color Combiner Inputs

When the color combiner stage executes, the system SHALL accept the following inputs per pass:

- **TEX_COLOR0 (CC_TEX0):** Sampled and decoded Q4.12 RGBA color from UNIT-011 sampler 0 (REQ-003.02)
- **TEX_COLOR1 (CC_TEX1):** Sampled and decoded Q4.12 RGBA color from UNIT-011 sampler 1 (REQ-003.02)
- **SHADE0 (CC_SHADE0):** Primary interpolated vertex color from the rasterizer (diffuse)
- **SHADE1 (CC_SHADE1):** Secondary interpolated vertex color from the rasterizer (specular)
- **CONST0 (CC_CONST0):** Per-draw-call constant color 0 (from CONST_COLOR[31:0])
- **CONST1 (CC_CONST1):** Per-draw-call constant color 1 / fog color (from CONST_COLOR[63:32])
- **DST_COLOR (CC_DST_COLOR):** Promoted destination pixel from the color tile buffer (RGB565 → Q4.12); available in all passes
- **COMBINED (CC_COMBINED):** Output of the previous combiner pass (pass 0 output feeding pass 1; pass 1 output feeding pass 2)
- **ONE (CC_ONE):** Constant 1.0
- **ZERO (CC_ZERO):** Constant 0.0

The RGB C slot additionally accepts alpha-to-RGB broadcast sources (e.g., TEX0 alpha, SHADE0 alpha) per INT-010.

### FR-009-2: Three-Pass Combiner Structure

When the firmware writes the CC_MODE register, the system SHALL configure three combiner passes evaluated in sequence per fragment using a single time-multiplexed datapath:

- Pass 0 (CC0): Computes `(A - B) * C + D` using the CC_MODE[31:0] input selectors
- Pass 1 (CC1): Computes `(A - B) * C + D` using the CC_MODE[63:32] input selectors; the COMBINED source is the pass 0 output
- Pass 2 (CC2/blend): Computes `(A - B) * C + D` using the CC_MODE[95:64] input selectors; the COMBINED source is the pass 1 output; DST_COLOR from the color tile buffer is available as an input source
- For rendering without blending, pass 2 SHALL be configured as a pass-through: A=COMBINED, B=ZERO, C=ONE, D=ZERO
- The RENDER_MODE.ALPHA_BLEND field selects a predefined pass-2 equation template (DISABLED, ADD, SUBTRACT, BLEND) per REQ-005.03; explicit CC_MODE[95:64] selectors override the template for custom configurations

### FR-009-3: Per-Component Operation

When the color combiner evaluates its equation, the system SHALL perform all operations independently per color component (R, G, B, A).
RGB and Alpha equations use independent input selector sets within each cycle.

### FR-009-4: Internal Precision

When the color combiner performs arithmetic, the system SHALL use Q4.12 signed fixed-point format (16-bit values) for all intermediate and output values, with saturation to the valid range per REQ-004.02.

### FR-009-5: Predefined Combiner Modes

When the firmware selects a combiner mode, the system SHALL support at minimum the following configurations expressible via CC_MODE:

- **MODULATE:** Pass 0: `TEX0 * SHADE0` (basic textured diffuse lighting)
- **DECAL:** Pass 0: `TEX0` (unlit texture)
- **LIGHTMAP:** Pass 0: `TEX0 * TEX1` (diffuse texture × pre-computed lightmap)
- **MODULATE_ADD (specular):** Pass 0: `TEX0 * SHADE0`; Pass 1: `COMBINED + SHADE1` (diffuse + specular)
- **FOG:** Pass 0: lit/textured color; Pass 1: `lerp(COMBINED, CONST1, SHADE0.A)` where SHADE0.A is fog factor
- **ALPHA_BLEND:** Pass 2: `COMBINED * COMBINED.A + DST_COLOR * (1 - COMBINED.A)` (Porter-Duff source-over using color tile buffer destination)
- **ADD blend:** Pass 2: `saturate(COMBINED + DST_COLOR)` (additive compositing)
- **SUBTRACT blend:** Pass 2: `saturate(COMBINED - DST_COLOR)` (subtractive compositing)

Additional combiner configurations are defined in INT-010.

## Verification Method

**Test:** Three VER documents together provide Test-level coverage for this requirement:

- **VER-004** (color_combiner_tb) — Verilator unit testbench that drives the `color_combiner.sv` module with controlled input vectors and compares outputs against a software reference model.
  Covers all three passes including pass-2 blend modes with a simulated DST_COLOR input.
  Note: UNIT-010 (Color Combiner) is marked Status: WIP.
  VER-004 implementation is deferred until UNIT-010 stabilization.
- **VER-013** (color-combined output golden image) — Pipeline integration test that renders a scene exercising multi-pass combiner modes through the full RTL hierarchy under Verilator.
  The rendered framebuffer is compared pixel-exactly against the approved golden image at `integration/golden/color_combined.ppm`.
  VER-013 remains blocked by UNIT-010 WIP status.
- **VER-024** (alpha blend modes golden image) — Integration test exercising pass-2 blend modes (ADD, SUBTRACT, BLEND) with the color tile buffer supplying DST_COLOR.

Acceptance criteria (verified jointly by VER-004, VER-013, and VER-024):

- CC_MODE register configures all three combiner passes via INT-010 encoding.
- TEX_COLOR0 and TEX_COLOR1 inputs are correctly sourced from UNIT-011 (Texture Sampler) Q4.12 outputs via the UNIT-006 pipeline bus.
- SHADE0 and SHADE1 inputs are correctly sourced from rasterizer interpolation outputs.
- CONST0 and CONST1 are correctly loaded from CONST_COLOR register (0x19).
- COMBINED in pass 1 equals pass 0 output; COMBINED in pass 2 equals pass 1 output.
- MODULATE mode produces `TEX0 * SHADE0` within Q4.12 rounding tolerance.
- DECAL mode passes TEX0 through unchanged (C=ONE, B=D=ZERO).
- LIGHTMAP mode produces `TEX0 * TEX1` within Q4.12 rounding tolerance.
- MODULATE_ADD mode: two-pass specular composition matches reference.
- FOG mode blends combined color toward CONST1 based on fog factor.
- Pass-2 ADD blend: `saturate(COMBINED + DST_COLOR)` matches reference.
- Pass-2 SUBTRACT blend: `saturate(COMBINED - DST_COLOR)` matches reference.
- Pass-2 BLEND: Porter-Duff source-over with COMBINED.A matches reference.
- DST_COLOR promoted from RGB565 to Q4.12 using MSB replication.
- All operations apply per-component (R, G, B, A independently).
- Pass-through (pass 2 with A=COMBINED, B=ZERO, C=ONE, D=ZERO) produces output equal to pass 1.

## Notes

This requirement supersedes the previous sequential texture blend model (TEX0 → TEX1 → TEX2 → TEX3).
The color combiner absorbs texture blending, vertex color modulation, and alpha blending into a single programmable three-pass structure.
The dedicated alpha blend unit (formerly downstream of the combiner) is retired; all blending is expressed as pass-2 CC equations using DST_COLOR from the color tile buffer.

All arithmetic uses Q4.12 signed fixed-point format per REQ-004.02.
The Q4.12 arithmetic constants ONE (0x1000) and ZERO (0x0000) are centralized in `rtl/pkg/fp_types_pkg.sv`.

The specific register encoding for combiner input selectors is defined in INT-010 (GPU Register Map), CC_MODE register.
