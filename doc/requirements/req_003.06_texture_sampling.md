# REQ-003.06: Texture Sampling

## Requirement

When the texture sampler (UNIT-011) processes a fragment during rasterization, the system SHALL perform an INDEXED8_2X2 palette lookup — fetching an 8-bit index from the index cache, extracting the sub-texel quadrant from the wrapped UV low bits, and reading the corresponding UQ1.8 RGBA color from the shared palette LUT — and produce a Q4.12 RGBA value for downstream processing.

## Rationale

This requirement defines the functional behavior of the texture sampling subsystem (UNIT-011) for the single supported format INDEXED8_2X2.
Each 2×2 apparent-texel tile stores one 8-bit palette index; the palette entry for that index holds four UQ1.8 RGBA colors (NW, NE, SW, SE) selected by the sub-texel quadrant `{v[0], u[0]}`.
The palette LUT (UNIT-011.06) is shared between both samplers; per-sampler index caches (UNIT-011.03) provide single-cycle index access.

## Functional Requirements

### FR-024-1: INDEXED8_2X2 Index Fetch

When sampling a fragment at apparent texel coordinates (u, v), UNIT-011 SHALL fetch the 8-bit palette index for the tile `(u >> 1, v >> 1)` from the per-sampler index cache (UNIT-011.03).
On an index cache miss the sampler SHALL stall the pipeline and fill the cache line from SDRAM before resuming.

### FR-024-2: Quadrant Extraction

After UV wrapping (REQ-003.05), UNIT-011.01 SHALL extract the 2-bit sub-texel quadrant as `quadrant[1:0] = {v_wrapped[0], u_wrapped[0]}`, where bit 1 selects the row (0 = north, 1 = south) and bit 0 selects the column (0 = west, 1 = east).

### FR-024-3: Palette LUT Lookup

UNIT-011.06 SHALL read the UQ1.8 RGBA color at address `{slot[0], idx[7:0], quadrant[1:0]}` from the shared 2-slot palette LUT, where `slot` is the value of TEXn_CFG.PALETTE_IDX for the active sampler.
Each palette slot holds 256 entries × 4 quadrant colors = 1024 UQ1.8 RGBA words.

### FR-024-4: NEAREST Filter

The sampler SHALL apply NEAREST filtering: the quadrant color read from the palette LUT is used directly as the output texel color.
No interpolation between palette entries is performed.
Filter mode is fixed at NEAREST (TEXn_CFG.FILTER = 2'd0); see INT-010.

### FR-024-5: UV Coordinate Wrapping

Before index and quadrant computation, UNIT-011.01 SHALL apply the wrap mode configured in TEXn_CFG.U_WRAP and TEXn_CFG.V_WRAP to the apparent (u, v) coordinates.
Wrapping is performed on the full apparent coordinates; the low bit of the wrapped result becomes the quadrant selector.
See REQ-003.05 for wrapping mode definitions.

### FR-024-6: Q4.12 Promotion

After palette LUT read, UNIT-011.06 SHALL promote each UQ1.8 channel to Q4.12 for the fragment pipeline:

```text
Q4_12 = {3'b0, uq1_8[8:0], 3'b0}
```

This maps UQ1.8 0x100 (= 1.0) to Q4.12 0x1000 (= 1.0) with no precision loss.
All four channels (R, G, B, A) use the same formula.

### FR-024-7: Cache-Aware Sampling Pipeline

The texture sampler (UNIT-011) SHALL execute the following pipeline for each fragment:

1. Apply UV wrapping (FR-024-5) to apparent (u, v) and extract quadrant (FR-024-2)
2. Compute index-cache set using XOR-folded addressing on `(u >> 1, v >> 1)` and compare tag
3. On cache hit: read 8-bit index from UNIT-011.03 in one cycle
4. On cache miss: stall pipeline, fetch the 4×4 index block (16 bytes) from SDRAM, fill cache line, then resume
5. Look up `{slot, idx, quadrant}` in the palette LUT (UNIT-011.06) and read UQ1.8 RGBA
6. Promote to Q4.12 (FR-024-6) and forward to the color combiner

Each of the 2 texture samplers has its own independent index cache (REQ-003.08).
The palette LUT is shared; both samplers address it via their respective PALETTE_IDX slot selections.

## Verification Method

**Test:** Execute the palette/index-cache unit testbench and relevant integration tests:

- [ ] INDEXED8_2X2 index fetch returns the correct 8-bit index for a known tile address
- [ ] Quadrant extraction `{v[0], u[0]}` selects the correct palette color (all 4 quadrant combinations)
- [ ] Palette slot 0 and slot 1 return independent colors for the same index
- [ ] NEAREST filter output equals the palette LUT entry exactly (no interpolation)
- [ ] UV wrapping applies before quadrant extraction; MIRROR mode swaps NE↔NW and SE↔SW correctly
- [ ] Index cache hit returns correct index in 1 cycle
- [ ] Index cache miss triggers SDRAM fetch and correct 16-byte cache line fill
- [ ] Cache invalidation on TEXn_CFG write prevents stale index data
- [ ] UQ1.8 value 0x100 promotes to Q4.12 value 0x1000
- [ ] Both samplers sample independently in a dual-texture draw call
- [ ] Minimum texture size 2×2 apparent texels (WIDTH_LOG2 ≥ 1, HEIGHT_LOG2 ≥ 1) samples correctly

## Design Allocation

- UNIT-011.01 (UV Coordinate Processing) — UV wrapping, quadrant extraction, index-resolution coordinate output (FR-024-2, FR-024-5)
- UNIT-011.03 (Index Cache) — per-sampler 8-bit index cache, SDRAM fill FSM (FR-024-1, FR-024-7)
- UNIT-011.06 (Palette LUT) — shared 2-slot palette, SDRAM load FSM, UQ1.8 storage, Q4.12 promotion (FR-024-3, FR-024-6)

## Notes

The INDEXED8_2X2 format stores one 8-bit index per 2×2 apparent-texel tile in SDRAM (INT-014).
Each palette entry for a given index holds four UQ1.8 RGBA colors in a PDPW16KD-backed LUT, addressed by `{slot[0], idx[7:0], quadrant[1:0]}`.
The palette must be loaded via PALETTEn.LOAD_TRIGGER before sampling; hardware does not fault on an unloaded palette.
See INT-014 for index array and palette blob addressing details, and UNIT-011.06 for the UNORM8→UQ1.8 promotion applied during palette load.
Texture colors produced by sampling are delivered as Q4.12 RGBA from UNIT-011 and feed into the color combiner stage (REQ-004.01) via UNIT-006.
