## Functional Requirements

### FR-051-1: FPGA Resource Budget

The GPU design SHALL fit within the following ECP5-25K resource budget:

| Resource | Budget | Allocation |
|----------|--------|------------|
| EBR (9kb blocks) | ≤39 of 56 (70%) | 6 texture subsystem (UNIT-011: 4 shared palette DP16KD + 2 per-sampler index DP16KD) + 8 Hi-Z metadata (UNIT-005: DP16KD, 9-bit min_z per tile using Z[15:7], sentinel 0x1FF = uninitialized, 4 entries per 36-bit word) + 1 Z-cache uninitialized flag array (UNIT-012: DP16KD, 32-bit wide, 512 addresses, 16,384 1-bit flags) + 4–7 color tile cache (UNIT-013: 2 × DP16KD data + 1–4 × PDPW16KD tags + 1 × DP16KD uninit flags) + 1 dither matrix + 2 color grade LUT (double-buffered) + 1 scanline FIFO + 1 SPI receive FIFO + 1 triangle setup reciprocal (`raster_recip_area.sv`, DP16KD 36×512) + 1 per-pixel 1/Q reciprocal (`raster_recip_q.sv`, DP16KD 18×1024); estimated total 25–29 blocks used |
| DSP slices (18x18) | ≤16 of 28 (57%) | 8 MULT18X18D in UNIT-005 (rasterizer): 2 for edge C-coefficient computation and sequential derivative inv_area scaling (time-shared, DD-036), 1 for triangle setup reciprocal interpolation, 1 for per-pixel 1/Q reciprocal interpolation, 4 for per-pixel perspective correction (S×1/Q and T×1/Q for two texture units); 3–4 MULT18X18D in UNIT-010 (Color Combiner) for Q4.12 multiply-accumulate across 3 time-multiplexed passes (CC0, CC1, CC2/blend); 0 MULT18X18D for texture subsystem (INDEXED8_2X2 palette lookup requires no multipliers); 0 MULT18X18D for UNIT-013 (color tile cache — tag compare and LRU logic use LUT4 only); total 11–12 of 16 budget |
| SERDES channels | 4 of 4 | DVI TMDS (3 data + 1 clock) |

### FR-051-2: Pipeline Timing

All pixel pipeline stages SHALL be fully pipelined with no throughput reduction:

- Depth range clipping: combinational comparison (0 additional cycles, ~20-40 LUTs for two 16-bit comparators)
- Early Z-test: reuses existing Z-buffer read port (0 additional cycles when bypassed; ~0 additional LUTs, reorders existing Z-test logic)
- Texture format promotion: combinational (0 additional cycles)
- Ordered dithering: 1 cycle EBR read (pipelined, no stall)
- Color grading LUT: 2 core clock cycle scanout latency (20 ns at 100 MHz, within the 40 ns pixel period)

## Verification Method

**Analysis:** Measure actual performance/resource usage against targets.

- [ ] EBR usage ≤39 blocks after synthesis (includes 6 texture EBR in UNIT-011, 8 Hi-Z DP16KD blocks in UNIT-005, 1 Z-cache uninitialized flag DP16KD in UNIT-012, and 4–7 color tile cache EBR in UNIT-013)
- [ ] DSP slice usage ≤16 after synthesis
- [ ] SERDES usage = 4 channels
- [ ] No pipeline throughput reduction from dithering or Q4.12 math
- [ ] Color grading LUT latency ≤2 cycles at scanout

## Notes

Non-functional requirement. See specifications for specific numeric targets.

EBR budget is 39 blocks, accommodating: texture subsystem (6 EBR, UNIT-011 — 4 shared palette DP16KD + 2 per-sampler index DP16KD), Hi-Z block metadata (8 EBR, UNIT-005, DP16KD storing 9-bit min_z per 4×4 tile using Z[15:7] with sentinel 0x1FF meaning uninitialized, 4 entries per 36-bit word), Z-cache uninitialized flag array (1 EBR, UNIT-012, DP16KD 32-bit wide storing 16,384 1-bit flags as 512 words, cleared in ~512 cycles), color tile cache (4–7 EBR, UNIT-013 — 2 × DP16KD data RAMs + 1–4 × PDPW16KD tag RAMs + 1 × DP16KD uninit-flag array), ordered dithering matrix (1 EBR, REQ-005.10), color grading LUT double-buffer (2 EBR, REQ-006.03 FR-133-4), display controller scanline FIFO (1 EBR, UNIT-008), SPI receive FIFO (1 EBR, UNIT-001), triangle setup reciprocal DP16KD (1 EBR, `raster_recip_area.sv`), and per-pixel 1/Q reciprocal DP16KD (1 EBR, `raster_recip_q.sv`).
Estimated total EBR usage is approximately 25–29 blocks, within the ≤39-block budget.
Post-synthesis verification is required to confirm the exact synthesis-tool EBR inference count, particularly the tag RAM count for UNIT-013 (1–4 PDPW16KD depending on tag width and synthesis mapping).
The scanline FIFO (1024×16 sync_fifo) infers as a DP16KD block RAM now that memory reads are separated from the async reset path.
DSP budget applies to both UNIT-005 (Rasterizer) and UNIT-010 (Color Combiner) fixed-point multipliers.
Q4.12 is 16 bits wide (4 integer + 12 fractional); 16×16 multiplication fits within the ECP5 18×18 DSP slice.
UNIT-005 uses 8 MULT18X18D: 2 for edge C-coefficient setup and sequential derivative inv_area scaling (time-shared across setup, edge evaluation, and derivative computation per DD-036), 1 for triangle setup reciprocal interpolation (`raster_recip_area.sv`), 1 for per-pixel 1/Q reciprocal interpolation (`raster_recip_q.sv`), and 4 dedicated multipliers for per-pixel perspective correction (U×(1/Q) and V×(1/Q) for sampler 0 and sampler 1).
With optional Newton-Raphson refinement enabled for inv_area, UNIT-005 uses 9 MULT18X18D.
UNIT-010 (Color Combiner) requires 3–4 DSP slices for the single time-multiplexed Q4.12 multiply-accumulate unit executing three passes (CC0, CC1, CC2/blend) per fragment.
The dedicated alpha blend unit (previously ~2–4 DSP slices) is retired; blending is now performed by CC pass 2 within UNIT-010.
Combined rasterizer and combiner DSP usage of 11–12 slices remains within the ≤16 budget, leaving 4–5 slices of margin.
The setup-iteration overlap FIFO (default depth 2) uses approximately 1460 flip-flops in fabric registers, not EBR.

**Texture subsystem EBR allocation (UNIT-011):** The palette LUT (UNIT-011.06) uses 4 DP16KD blocks (1024×18 mode, 2 in parallel per slot to form a 1024×36 dual-port memory) shared between both samplers, addressed by `{idx[7:0], quadrant[1:0]}` per slot.
Each per-sampler index cache (UNIT-011.03) uses 1 DP16KD block in 1024×18 mode storing 8-bit indices in a direct-mapped 32-set × 16-indices/line layout, with two adjacent indices packed per EBR word (8 EBR words per cache line — see UNIT-011.03 §EBR Notes).
The 1024×18 packing matches the SDRAM 16-bit word width so that the streaming fill writes one EBR word per arriving SDRAM burst cycle, allowing the cache to be inferred onto a single DP16KD instead of falling back to LUT-based distributed RAM.
Total texture EBR: 4 (shared palette) + 2 (per-sampler index caches) = 6 EBR.
LUT4 estimate for the texture subsystem is approximately 750 LUT4; 0 DSP slices are required (palette lookup is an EBR read with no arithmetic).
The color combiner module reuses the same DSP slices for Q4.12 multiply-accumulate; no additional DSP resources are required for the texture path.

**Color tile cache resource estimate (UNIT-013):** UNIT-013 uses 4–7 EBR blocks: 2 × DP16KD for the 4-way × 32-set data RAM (256-bit cache line per way), 1–4 × PDPW16KD for tag storage (tag width and synthesis mapping determine the exact count), and 1 × DP16KD for the per-tile uninitialized-flag array (parallel to the Z-cache uninit-flag EBR in UNIT-012).
The cache FSM, LRU state, and tag-compare logic require approximately 250 LUT4 and 0 DSP slices.

The command FIFO (UNIT-002) uses distributed RAM (LUTs) rather than EBR.
The 72-bit × 512-deep command FIFO consumes approximately 36,864 bits of distributed RAM, which is accounted for in the LUT budget rather than the EBR budget.
The 512-entry depth was chosen to absorb a full frame of GPU commands without stalling the host, amortizing the SPI transaction overhead over longer bursts.

**SDRAM controller resource estimate (UNIT-007, SDRAM controller):** The SDRAM controller (sdram_controller.sv) requires approximately 200-300 LUTs and 100-150 FFs for the 8-state FSM (INIT, IDLE, ACTIVATE, READ, WRITE, PRECHARGE, REFRESH, DONE), address decomposition (byte address to bank/row/column), CAS latency pipeline (3-stage shift register), refresh timer (14-bit counter for 781-cycle interval), initialization sequence counter (15-bit for 200 us wait), burst counter (8-bit burst_remaining), preemption logic, and data routing.
This is larger than the original ~150 LUT async SRAM controller because SDRAM requires initialization, auto-refresh, and explicit row activation/precharge management.
No additional EBR or DSP resources are required.

**PLL resource note:** The SDRAM controller requires a 90-degree phase-shifted 100 MHz clock output from the ECP5 PLL.
The ECP5 EHXPLLL primitive supports up to 4 clock outputs with configurable phase shift.
The existing PLL already uses 3 outputs (clk_core 100 MHz, clk_pixel 25 MHz, clk_tmds 250 MHz).
Adding the SDRAM clock as the 4th output consumes the last available PLL output but requires no additional PLL primitives.
