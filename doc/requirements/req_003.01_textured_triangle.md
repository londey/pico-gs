# REQ-003.01: Textured Triangle

## Requirement

When the firmware writes three VERTEX registers with RENDER_MODE configured for texture mapping, the system SHALL rasterize a triangle with UV coordinates interpolated across the surface and sample texture data at each pixel using the configured texture unit.

## Rationale

Texture mapping is essential for realistic rendering, allowing surface detail without increasing geometric complexity. It enables: realistic materials (wood, metal, stone), decals and labels, sprite rendering for UI, and efficient detail representation (one 256×256 texture = 65,536 triangles of flat-shaded detail).

## Verification Method

**Test:** Two VER documents provide primary Test-level coverage for this requirement, with a third providing supplementary integration coverage:

- **VER-012** (textured triangle golden image) — Pipeline integration test that renders a textured triangle through the full RTL hierarchy under Verilator, using a behavioral SDRAM model populated with a known INDEXED8_2X2 test texture and palette.
  The rendered framebuffer is compared pixel-exactly against the approved golden image at `integration/golden/textured_triangle.ppm`.
- **VER-014** (textured cube golden image) — Supplementary integration test that renders a perspective-projected textured cube through the full RTL pipeline.
  Exercises perspective-correct UV interpolation more thoroughly than VER-012 because the cube's faces subtend a wide range of projection angles, producing stronger foreshortening in the UV gradients.
  Also exercises depth-ordered multi-triangle rendering (REQ-005.02) simultaneously.
- The palette/index-cache unit testbench provides component-level coverage of UNIT-011.03 and UNIT-011.06 in isolation.

Acceptance criteria (verified jointly by VER-012; supplementally by VER-014):

- TEXn_CFG.ENABLE, FORMAT (= INDEXED8_2X2 = 4'd0), BASE_ADDR, WIDTH_LOG2, HEIGHT_LOG2, and PALETTE_IDX configure the texture unit correctly.
- UV coordinates (U/W, V/W, 1/W) are perspective-correct; no affine warping is visible. UV values on the rasterizer→pixel_pipeline fragment bus use Q4.12 format per component.
- Texture coordinates wrap or clamp per TEXn_CFG.U_WRAP / V_WRAP.
- Final pixel equals texture color × vertex color (MODULATE combiner mode).

## Notes

Texture mapping requires both geometric data (vertices) and UV coordinates.
The GPU supports up to 2 simultaneous texture units (REQ-003.02) with a programmable color combiner (REQ-004.01).
