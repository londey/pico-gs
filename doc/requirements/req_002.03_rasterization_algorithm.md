# REQ-002.03: Rasterization Algorithm

## Classification

- **Priority:** Essential
- **Stability:** Stable
- **Verification:** Test

## Requirement

When a triangle is submitted via the vertex registers and a VERTEX_KICK write is issued, the system SHALL compute edge function coefficients and a screen-clamped bounding box (UNIT-004), then iterate over all pixels within the bounding box in 4×4 tile order (UNIT-005), emitting per-fragment data (x, y, z, color0, color1, uv0, uv1, lod) to the pixel pipeline (UNIT-006) via a valid/ready handshake for each pixel that passes the edge function inside test.

When the rasterizer emits fragments at the 100 MHz core clock rate, the system SHALL sustain a peak throughput of one fragment evaluation per clock cycle, yielding up to 100 million fragment evaluations per second.

## Rationale

Edge-function rasterization with 4×4 tile-order traversal provides deterministic coverage evaluation aligned to the SDRAM block-tiled surface layout.
Tile order maximizes Z-buffer tile cache locality and enables burst-coalesced framebuffer writes: all pixels within a 4×4 tile are processed before advancing to the next tile, matching the surface tiling format used by the Z-buffer cache and write-coalescing buffer (see UNIT-007 and INT-011).
Operating at 100 MHz (unified GPU core clock), the rasterizer achieves double the throughput compared to a 50 MHz design, enabling higher fill rates for complex scenes.
The rasterizer emits a structured fragment bus to UNIT-006 (Pixel Pipeline), which owns framebuffer and Z-buffer SDRAM writes.
See UNIT-004 (Triangle Setup), UNIT-005 (Rasterizer), and UNIT-006 (Pixel Pipeline) for implementation details.

## Notes

Functional requirements grouped from specification.

**Throughput**: At 100 MHz core clock, the rasterizer (UNIT-005) evaluates one pixel per cycle in the inner loop.
Effective fill rate is limited by SDRAM bandwidth contention with display scanout and Z-buffer operations (see INT-011 bandwidth budget).
The theoretical peak of 100 Mpixels/sec evaluations translates to approximately 25 Mpixels/sec sustained write throughput after arbitration overhead.

**Incremental interpolation**: The rasterizer uses precomputed attribute derivatives (dAttr/dx, dAttr/dy) from UNIT-004 to step attribute values within and across 4×4 tiles via addition only, replacing per-pixel barycentric multiply-accumulate.
This substantially reduces DSP usage compared to the barycentric MAC approach, creating headroom for the per-pixel perspective correction multipliers.
See UNIT-005 for the derivative precomputation and stepping implementation.

**Perspective-correct texture coordinates**: The host submits texture coordinates pre-divided by W (S=U/W, T=V/W in Q4.12) along with Q=1/W (also Q4.12) per vertex via the UV0_UV1 and VERTEX registers.
The rasterizer interpolates S, T, Q linearly across the triangle and, per pixel, multiplies S and T by 1/Q to recover true perspective-correct U and V coordinates.
The fragment bus carries true U, V coordinates (Q4.12), not the raw S, T values.
Q is consumed internally and does not appear on the fragment bus.

**Screen-space linear colors and depth**: COLOR0 and COLOR1 are interpolated linearly in screen space (no perspective correction), as an explicit design choice (see REQ-002.02 rationale).
Z is also interpolated linearly in screen space.

**Fragment output bus**: The rasterizer emits fragments in 4×4 tile order to UNIT-006 via a valid/ready handshake.
Each fragment carries x, y, z, color0, color1, uv0, uv1, and lod.
Format annotations: color0 and color1 are Q4.12 RGBA (64 bits each); uv0 and uv1 carry true perspective-correct U, V coordinates in Q4.12 (U[31:16], V[15:0] packed into 32 bits each); lod is UQ4.4 (8 bits: 4-bit integer mip level, 4-bit trilinear blend fraction); z is a 16-bit unsigned depth value.
UNIT-006 (Pixel Pipeline) is responsible for texture fetch, color combining, alpha blending, and all SDRAM writes (framebuffer and Z-buffer).
See UNIT-007 for arbiter burst mode handling of framebuffer and Z-buffer writes with the 4×4 block-tiled layout (INT-011).
