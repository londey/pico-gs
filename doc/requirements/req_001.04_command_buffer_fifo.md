# REQ-001.04: Command Buffer FIFO

## Classification

- **Priority:** Essential
- **Stability:** Stable
- **Verification:** Test

## Requirements

### REQ-001.04.1: Command FIFO Buffering

When the host submits a write transaction via SPI, the system SHALL enqueue the command into a 32-entry deep asynchronous FIFO that bridges the SPI clock domain to the 100 MHz GPU core clock domain, preserving strict submission order.

### REQ-001.04.2: FIFO Full Flow Control

When the command FIFO occupancy reaches DEPTH-2 (30) entries, the system SHALL assert the CMD_FULL GPIO signal (active high) to indicate that the host should pause write transactions.

### REQ-001.04.3: FIFO Empty Indication

When the command FIFO contains zero pending entries and no command is executing, the system SHALL assert the CMD_EMPTY GPIO signal (active high).

### REQ-001.04.4: FIFO Overflow Protection

When the host submits a write transaction while the command FIFO is full, the system SHALL discard the command silently (no error signaling).

### REQ-001.04.5: Autonomous Boot Screen

After power-on reset, the GPU SHALL autonomously render and display a boot screen without requiring any SPI traffic from the host.
The boot screen SHALL consist of a black background with a Gouraud-shaded triangle (red, green, blue vertex colors) centered on the display.

### REQ-001.04.6: Boot Completion Before Host

When the host firmware begins SPI communication, the boot sequence SHALL have completed, leaving the command FIFO empty and ready to accept host commands.

## Rationale

The command FIFO decouples the SPI clock domain from the GPU core clock domain (100 MHz), allowing the host to submit commands without blocking on GPU execution.
The SPI-to-core clock domain crossing remains asynchronous since SPI clock frequency is independent of the GPU core clock.
The autonomous boot screen provides a visual self-test confirming that the FPGA bitstream loaded correctly and the GPU rendering pipeline is functional, without requiring host firmware participation.

## Verification Method

**Test:** For each requirement:
- REQ-001.04.1: Submit write transactions via SPI and confirm commands appear in FIFO in order, with FIFO depth of 32.
- REQ-001.04.2: Fill FIFO to 30 entries and confirm CMD_FULL asserts; drain below 30 and confirm deasserts.
- REQ-001.04.3: Drain FIFO completely and confirm CMD_EMPTY asserts; submit a write and confirm deasserts.
- REQ-001.04.4: Fill FIFO to 32 entries, submit additional write, confirm it is discarded with no error.
- REQ-001.04.5: After power-on, capture framebuffer contents and confirm black background with centered RGB Gouraud triangle, without any SPI traffic from the host.
- REQ-001.04.6: Wait 100 ms after power-on, confirm CMD_EMPTY is asserted (boot sequence complete, FIFO ready for host commands).

## Notes

The FIFO read-side clock is the unified 100 MHz GPU core clock (clk_core).
The FIFO write-side clock is the SPI clock domain; this crossing remains asynchronous.
