# REQ-003.09: Texture Palette Slots

## Requirement

When a texture unit is enabled and TEXn_CFG.PALETTE_IDX selects palette slot N, the system SHALL look up the sampled 8-bit index and sub-texel quadrant in palette slot N to produce the UQ1.8 RGBA texel color delivered to the color combiner.

## Rationale

The INDEXED8_2X2 format separates the index array (stored in SDRAM per-texture) from the color codebook (held in shared on-chip EBR palette slots), enabling compact index storage while keeping texel color data on-chip for single-cycle lookup once the index is available.

## Functional Requirements

### FR-003.09-1: Resident Palette Slots

When the GPU is operational, the palette subsystem SHALL maintain two resident palette slots (slot 0 and slot 1), each holding 256 entries × 4 UQ1.8 RGBA quadrant colors (NW, NE, SW, SE), for a total of 1024 UQ1.8 RGBA values per slot.

### FR-003.09-2: Palette Slot Selection

When sampling a texture fragment, the system SHALL select the palette slot used for the lookup from the TEXn_CFG.PALETTE_IDX[24:24] field of the active texture unit.

### FR-003.09-3: Quadrant-Based Lookup

When performing a palette lookup for a fragment, the system SHALL select the quadrant color from the palette entry addressed by the 8-bit index using `quadrant[1:0] = {v_wrapped[0], u_wrapped[0]}`, where bit 1 selects the V axis (0 = North, 1 = South) and bit 0 selects the U axis (0 = West, 1 = East).

### FR-003.09-4: Firmware-Triggered SDRAM Load

When firmware writes a 1 to PALETTE0.LOAD_TRIGGER or PALETTE1.LOAD_TRIGGER, the system SHALL initiate an SDRAM burst read starting at the address encoded in the corresponding PALETTEn.BASE_ADDR field (byte address = BASE_ADDR × 512), promote each RGBA8888 byte to UQ1.8 inline, and store the result into the selected palette slot's on-chip EBR.

### FR-003.09-5: Self-Clearing Trigger

When the SDRAM load initiated by a PALETTEn.LOAD_TRIGGER write completes, the system SHALL clear the LOAD_TRIGGER bit automatically with no firmware write required.

### FR-003.09-6: Per-Slot Load Isolation

When a SDRAM load of palette slot N is in progress, the system SHALL continue to serve texture sampling reads from palette slot M (where M ≠ N) without stalling.

### FR-003.09-7: In-Flight Index Fill Preemption

When an index cache fill request arrives while a palette load is pending on the SDRAM arbiter port, the system SHALL allow the index cache fill to preempt the pending palette load request in the internal arbiter so that fragment rendering is not delayed unnecessarily.

### FR-003.09-8: No Hardware Status Readback

When firmware loads a palette slot, the system SHALL provide no hardware status register indicating load completion; firmware SHALL track load completion by cycle-counting or by serializing palette load writes before dependent triangle submissions.

### FR-003.09-9: Undefined Reset State

When the GPU comes out of reset, the contents of both palette slots SHALL be undefined; firmware SHALL perform a PALETTEn.LOAD_TRIGGER write to initialize each slot before submitting triangles that reference that slot.

### FR-003.09-10: Shared Palette EBR

When both texture units (TEX0 and TEX1) are active, the two palette slots SHALL be stored in a single shared EBR array accessible by both samplers, with no per-sampler palette duplication.

## Verification Method

**Test:** Verify palette slot behavior against the following acceptance criteria:

- [ ] A PALETTEn.LOAD_TRIGGER write causes SDRAM reads from BASE_ADDR × 512 covering 4096 bytes (256 entries × 4 RGBA8888 colors)
- [ ] Each UNORM8 byte in the SDRAM payload is promoted to UQ1.8 correctly on load (0x00 → 0x000, 0xFF → 0x100)
- [ ] After load, sampling with index I and quadrant Q returns the correct UQ1.8 RGBA color from entry I, quadrant Q
- [ ] TEXn_CFG.PALETTE_IDX = 0 selects slot 0; PALETTE_IDX = 1 selects slot 1
- [ ] Quadrant bit 1 tracks V_wrapped[0] (North/South), bit 0 tracks U_wrapped[0] (West/East)
- [ ] LOAD_TRIGGER self-clears after load completes
- [ ] Reads from slot M are unaffected while slot N is being loaded (N ≠ M)
- [ ] Index cache fill preempts a pending palette load in the internal arbiter
- [ ] TEX0 and TEX1 both access the same shared palette EBR without duplication

## Design Allocation

- UNIT-011.06 (Palette LUT) — shared 2-slot EBR codebook (4 PDPW16KD), SDRAM load FSM, inline UNORM8→UQ1.8 promotion, quadrant addressing `{slot, idx[7:0], quadrant[1:0]}`, per-slot isolation

## Notes

The palette blob in SDRAM contains 256 entries × 4 RGBA8888 colors = 4096 bytes per slot, addressed via PALETTEn.BASE_ADDR × 512.
UNORM8→UQ1.8 promotion is performed inline during the SDRAM load burst; no promotion hardware is needed in the sampling hot path.
See INT-010 for PALETTE0/PALETTE1 register field definitions and INT-014 for the SDRAM palette blob layout.
The 4096-byte SDRAM load requires approximately 515 burst cycles at the 16-bit SDRAM bus width (2048 16-bit words, amortized over row activation overhead).
