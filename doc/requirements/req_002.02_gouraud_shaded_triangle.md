# REQ-002.02: Gouraud Shaded Triangle

## Classification

- **Priority:** Important
- **Stability:** Stable
- **Verification:** Demonstration

## Requirement

When the firmware writes three VERTEX registers with RENDER_MODE configured for Gouraud shading (GOURAUD=1), setting a distinct COLOR register value before each VERTEX write, the system SHALL interpolate color linearly across the triangle in screen space and write the interpolated RGBA8 color to each covered pixel.

## Rationale

Gouraud shading enables smooth per-vertex lighting gradients by linearly interpolating per-vertex colors across triangle surfaces.
This is the primary shading mode for diffuse lighting effects and smooth color transitions.

**Design choice — screen-space linear interpolation:** Color attributes (COLOR0 and COLOR1) are interpolated linearly in screen space, without perspective correction.
This is an intentional decision (see Technical Report: Comprehensive Rasterizer Plan, Finding 7), not a limitation.
The visual difference between screen-space and perspective-correct color interpolation is subtle for Gouraud shading; many classic GPUs use screen-space interpolation for colors without perceptible artifacts.
This choice keeps the register interface simple (UNORM8 colors, no C/W pre-multiplication), eliminates eight per-pixel divisions from the pixel pipeline (four color channels × two color sets), and reduces rasterizer setup cost.
Texture coordinates use perspective-correct interpolation via the Q (1/W) denominator; only colors and depth are screen-space linear (see REQ-002.03).

## Verification Method

**Demonstration:** The system SHALL meet the following acceptance criteria:

- [x] Set RENDER_MODE with GOURAUD=1
- [x] Set COLOR register before each VERTEX write
- [x] Colors interpolate linearly across triangle in screen space (RGBA8)
- [ ] With dithering enabled, smooth gradients visible in RGB565 framebuffer output
- [ ] With dithering disabled, quantization artifacts limited to RGB565 precision
